重要教训：API 优先验证 Important Lesson: API Verification First

今天深刻学到一个重要教训：涉及 API 的项目应该先跑通 API，再进行后续开发。xAI API 看似免费，实际需要购买 credits，浪费了 6 次测试和大量时间。未来开发 API 集成项目时，必须先用最小化代码验证 API 可用性和免费额度。 Today learned an important lesson: projects involving APIs should verify the API first before development. xAI API appeared free, but actually required purchasing credits, wasting 6 tests and significant time. When developing API integration projects in the future, must verify API availability and free quota with minimal code first.
